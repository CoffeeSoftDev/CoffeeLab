🎯1. Introducción

El Sistema de Contabilidad CoffeeSoft es una solución integral diseñada para la gestión financiera y contable de empresas. Este manual técnico proporciona información detallada sobre la arquitectura, funcionalidades, configuración y mantenimiento del sistema.

Características Principales

  • Arquitectura MVC: Separación clara entre modelo, vista y controlador
  • Gestión de Sobres: Control centralizado de documentos contables
  • Control de Turnos: Seguimiento operacional por períodos
  • Gestión Financiera: Ingresos, compras, pagos y facturación
  • Administración de Terceros: Clientes y proveedores
  • Sistema de Archivos: Gestión documental integrada

🏗️2. Arquitectura del Sistema

Estructura MVC

Modelo-Vista-Controlador (MVC): El sistema sigue el patrón arquitectónico MVC para garantizar la separación de responsabilidades y facilitar el mantenimiento.

📁 Estructura de Directorios

contabilidad/ ├── index.php # Punto de entrada principal ├── caratula.php # Vista de carátula ├── consultas.php # Vista de consultas ├── sobres.php # Vista de gestión de sobres ├── ctrl/ # Controladores │ ├── _Apertura.php │ ├── _Archivos.php │ ├── _Caratula.php │ ├── _Clientes.php │ ├── _Compras.php │ ├── _Facturas.php │ ├── _Ingresos.php │ ├── _Pagos.php │ ├── _Sobres.php │ └── _Turnos.php ├── mdl/ # Modelos │ ├── mdl-pagos.php │ └── mdl-turno.php ├── layout/ # Plantillas │ ├── head.php │ └── script.php └── src/ # Recursos estáticos ├── css/ └── js/

Flujo de Datos

  1. Vista (View): Interfaz de usuario que presenta la información
  2. Controlador (Controller): Procesa las solicitudes y coordina las operaciones
  3. Modelo (Model): Gestiona los datos y la lógica de negocio
  4. Base de Datos: Almacenamiento persistente de información

🔐4. Sistema de Permisos

⚠️ Importante: El sistema implementa un control de acceso basado en roles y permisos específicos para cada módulo.

Niveles de Acceso

Nivel Descripción Permisos
Administrador Acceso completo al sistema Crear, Leer, Actualizar, Eliminar
Supervisor Acceso a módulos específicos Crear, Leer, Actualizar
Operador Acceso limitado a operaciones Crear, Leer
Consulta Solo visualización Leer

Validaciones de Seguridad

  • Validación de sesión activa
  • Verificación de permisos por módulo
  • Control de acceso a funciones específicas
  • Registro de actividades (audit trail)

🔧6. Solución de Problemas Comunes

Error de Conexión a Base de Datos

Síntoma: "No se puede conectar a la base de datos"

Solución:

  1. Verificar configuración en _Conect.php
  2. Comprobar que el servicio MySQL esté activo
  3. Validar credenciales de acceso
  4. Revisar permisos de usuario de base de datos

Errores de Permisos

Síntoma: "Acceso denegado" o "Permisos insuficientes"

Solución:

  1. Verificar sesión de usuario activa
  2. Comprobar permisos asignados al rol
  3. Revisar configuración de permisos por módulo
  4. Validar integridad de tabla de permisos

Problemas de Rendimiento

Síntoma: Lentitud en carga de datos o timeouts

Solución:

  1. Optimizar consultas SQL
  2. Revisar índices de base de datos
  3. Implementar paginación en listados grandes
  4. Considerar cache de consultas frecuentes

📊7. Reportes Disponibles

Reportes Financieros

  • Estado de Resultados: Ingresos vs gastos por período
  • Balance General: Situación financiera de la empresa
  • Flujo de Caja: Movimientos de efectivo
  • Cuentas por Cobrar: Cartera de clientes
  • Cuentas por Pagar: Obligaciones con proveedores

Reportes Operacionales

  • Resumen de Turnos: Actividad por período operacional
  • Análisis de Compras: Gastos por categoría y proveedor
  • Seguimiento de Pagos: Estado de pagos programados
  • Actividad de Usuarios: Log de acciones por usuario

Exportación de Datos

Los reportes pueden exportarse en los siguientes formatos:

  • PDF para impresión
  • Excel para análisis
  • CSV para integración
  • JSON para APIs
